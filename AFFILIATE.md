# X高単価アフィリエイト 送客システム

Xで集客し、高単価スクールの「無料面談」へ送客してアフィリエイト報酬を得るための
**指揮官（Commander）＋ 10エージェント** 構成の素材自動生成システムです。

案件名を入れるだけで、以下10フローの成果物一式を生成し、**Googleスプレッドシートの
案件ごとの新しいタブ**へ書き込みます。Xへの投稿・画像作成は手動（コピペ運用）です。

```
案件名入力 → 指揮官が案件を分析（◯◯を案件に合わせて自動補完）
   → 各エージェントが①〜⑩を実行 → 指揮官が信憑性・完成度を判定し必要なら再生成
   → 重複を除去 → スプレッドシートの新タブに整形して書き込み
```

## 生成される10フロー

| # | フロー | 出力 |
|---|---|---|
| ① | 売れるジャンル選定 | ジャンル20候補（適合度スコア付き・指揮官がベスト選定） |
| ② | ターゲット設計 | ターゲット10パターン（適合度スコア付き） |
| ③ | アカウントコンセプト | コンセプト10案 |
| ④ | 見込み客を集めるX投稿 | 投稿30本（**コピペ用** ＋ ChatGPT画像プロンプト） |
| ⑤ | 無料面談へ誘導する固定ポスト | 10本（コピペ用 ＋ 画像プロンプト） |
| ⑥ | DM誘導 | 5段階のDMテンプレート |
| ⑦ | 教育投稿 | 20本（コピペ用 ＋ 画像プロンプト） |
| ⑧ | オファー文改善 | 名称/キャッチ/導線/各SNS短文 |
| ⑨ | 見込み客診断ルブリック | S/A/B/C判定の運用マニュアル |
| ⑩ | 30日ロードマップ | 1〜30日の日次タスク |

- **指揮官**：全フローの出力を「信憑性（誇大・コンプラ違反がないか）」と「完成度（網羅性・実用性）」で採点。
  基準未満は1度だけ再生成し、総合所見をスプレッドシート冒頭に記載します。
- **◯◯の自動補完**：案件名から最適なジャンル・ターゲット・無料面談内容を推定。詳細を手入力すればそれを優先。
- **重複防止**：過去に生成した投稿文を `_dedup` タブに蓄積し、次回以降は被らない切り口で生成します。
- **Xは手動投稿**：投稿文はコピペ用列に並びます。画像は「画像プロンプト」をChatGPTに貼って手動作成します。

## セットアップ

### 1. Groq APIキー（必須・無料枠あり）
[console.groq.com](https://console.groq.com) でキーを取得し、`.env` または GitHub Secrets の `GROQ_API_KEY` に設定。

### 2. Googleスプレッドシート連携
1. [Google Cloud Console](https://console.cloud.google.com) で **Google Sheets API** を有効化。
2. **サービスアカウント**を作成し、JSONキーをダウンロード。
3. 出力用のスプレッドシートを作成し、サービスアカウントのメールアドレス（`xxx@xxx.iam.gserviceaccount.com`）に**編集権限で共有**。
4. スプレッドシートの ID（URLの `/d/` と `/edit` の間）を `AFFILIATE_SHEET_ID` に設定。
5. 認証情報を設定：
   - ローカル：JSONを `google-credentials.json` として置き、`GOOGLE_APPLICATION_CREDENTIALS=./google-credentials.json`
   - GitHub Actions：JSONの中身をそのまま Secret `GOOGLE_SERVICE_ACCOUNT_JSON` に貼り付け

> 未設定でも動作します。その場合は `data/affiliate/output/` にJSONとTSV（スプレッドシートに貼れる形式）を出力します。

## 使い方

### A. Web UI（スマホ・タブレット対応）
```bash
npm run affiliate         # http://localhost:3100
```
案件名を入力 → 「生成開始」。進捗がリアルタイム表示され、完了後にスプレッドシートのリンクが出ます。
詳細（価格・報酬・面談内容・目標など）を開いて入力すると、◯◯がその案件に合わせて変動します。

### B. CLI
```bash
npm run affiliate:run -- "AI学習スクール"
# または
AFFILIATE_OFFER="英語コーチング" npm run affiliate:run
```

### C. GitHub Actions（サーバー不要・スマホから実行）
GitHubアプリ → Actions → **「X送客 キャンペーン生成」** → Run workflow → 案件名を入力。
結果はスプレッドシートの新しいタブに書き込まれます（要 Secrets: `GROQ_API_KEY`, `AFFILIATE_SHEET_ID`, `GOOGLE_SERVICE_ACCOUNT_JSON`）。

## 案件の追加・管理（融通がきく設計）
案件は `data/affiliate/offers.json` に保存されます。Web UIの「案件情報を保存」、CLI、または
直接JSONを編集して、いくらでも案件を追加できます。各項目（ジャンル・価格・報酬・面談内容・
フォロワー数・目標送客数/報酬）は任意で、空欄ならAIが推定します。

## 運用フロー（人間の作業）
1. 案件名を入れて生成 → スプレッドシートの新タブを開く
2. 「投稿文（コピペ用）」をコピーしてXに投稿
3. 同じ行の「画像プロンプト（ChatGPT用）」をChatGPTに貼って画像を作成し、投稿に添付
4. DM・固定ポスト・オファー短文も同様にコピペで運用
5. 「投稿状況」列を手動更新して進捗管理

## 注意（コンプライアンス）
「必ず稼げる」等の断定・収入/効果保証は景表法・特商法に抵触します。指揮官が信憑性スコアで
チェックしますが、最終的な投稿可否は必ず人間が確認してください。
